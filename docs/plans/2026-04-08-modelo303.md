# Modelo 303 (IVA) Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Add support for AEAT Modelo 303 (IVA trimestral) to `boeidl` by extending the DSL to allow multiple `record` blocks per `.boe` file, adding an `unsigned_amount` field type, and compiling a full `models/mod303.boe` describing the 5 page layouts of `DR303e21v200.xlsx`.

**Architecture:** Scope C from brainstorm — minimal DSL extension. One `.boe` file can now contain N `record "name" { record_length = ... }` blocks; each block owns its own `field` / `derive` / `check` list. Codegen emits one Rust struct per record (e.g. `Mod303Page1`, `Mod303Page2`, ...). Envelope wrapping (`<T3030...>` header, `<AUX>` block, trailer) and page composition/repetition are **explicitly out of scope** for this PR — users wire pages together in handwritten Rust. A follow-up PR will add envelope + `repeat` + `complementary`. Pages 2 and 5 have a `Comp` (complementaria) column in the XLSX; for now we include every field and ignore the Comp flag (noted in a TODO comment at the top of mod303.boe).

**Tech Stack:** Same as existing boeidl — pest grammar, hand-written pest→AST, validator, string-templated Rust codegen. Existing mod130 tests must remain green (backwards compatibility).

**Source of truth:** `/tmp/m303/DR303.xlsx` (downloaded from sede.agenciatributaria.gob.es). Extracted field tables live in `/tmp/m303/fields.json` (generated via openpyxl) for reference when writing mod303.boe. Page lengths measured from XLSX: página 1 = 1464, página 2 = 1706, página 3 = 1079, página 4 = 986, página 5 = 1523.

**Working branch:** `feat/modelo303` (already created, off `main`).

## Task 0: Decide backwards-compatibility posture

**Context:** Today `BoeFile` has a flat `fields: Vec<Field>` / `derives` / `checks`, and the grammar's top-level is `model_block ~ (field_block | derive_stmt | check_block)*`. We need multiple records without breaking mod130.

**Decision:** Keep the flat top-level as **syntactic sugar for a single implicit record named after the model number** (`mod130.boe` stays untouched). Introduce a new optional `record "name" { ... }` block; when at least one `record` block is present, top-level `field`/`derive`/`check` are a parse error. `BoeFile` gains `records: Vec<Record>` where each `Record` owns `name`, `record_length`, `fields`, `derives`, `checks`. On the legacy path, the parser synthesizes a single `Record { name: format!("mod{}", model.number), record_length: model.record_length, ... }`. Codegen iterates over `records` and emits one struct per record using the existing logic.

**No code changes in this task** — this section just documents the invariant the following tasks enforce.

/// Encode an unsigned fixed-point amount as `width` ASCII digits, zero-padded.
/// `value` is the integer-cents representation (e.g. 1_000_000 with decimals=2 means 10000.00).
/// Errors on negative values or overflow.
pub fn encode_unsigned_amount(field: &str, value: i64, width: usize, _decimals: u8) -> Result<String, AeatError> {
    if value < 0 {
        return Err(AeatError::InvalidValue {
            field: field.to_string(),
            value: value.to_string(),
        });
    }
    let s = value.to_string();
    if s.len() > width {
        return Err(AeatError::FieldOverflow {
            field: field.to_string(),
            width,
            got: s.len(),
        });
    }
    Ok(format!("{:0>width$}", s, width = width))
}
```

Note: `_decimals` is accepted for symmetry with `encode_signed_amount` and future-proofing, even though the current encoding doesn't need it (decimals are implied by width — caller stores integer cents).

**Step 5:** In `runtime/bytes.rs`, add `parse_unsigned_amount`:

/// Parse `len` bytes at `at` (1-indexed) as an unsigned fixed-point amount.
/// Reads ASCII digits, leading zeros allowed. Returns i64 of integer cents.
pub fn parse_unsigned_amount(data: &[u8], at: usize, len: usize) -> Result<i64, AeatError> {
    let start = at - 1;
    let end = start + len;
    if end > data.len() {
        return Err(AeatError::ShortRecord { expected: end, got: data.len() });
    }
    let slice = &data[start..end];
    let s = std::str::from_utf8(slice).map_err(|_| AeatError::InvalidEncoding)?;
    let trimmed = s.trim_start_matches(|c: char| c == '0' || c == ' ');
    if trimmed.is_empty() { return Ok(0); }
    trimmed.parse::<i64>().map_err(|_| AeatError::InvalidValue {
        field: format!("parse_unsigned_amount@{at}"),
        value: s.to_string(),
    })
}
```
